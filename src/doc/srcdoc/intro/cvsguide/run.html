So you have just downloaded the source code, and you want to run OpenKore. To do that, download one of the following files:

  • OpenKore Win32 launcher (670 KB)
  • OpenKore Win32 launcher, with support for Wx interface (2,29 MB)
Extract the file to the same folder as openkore.pl.

To run OpenKore, run start.exe. To run OpenKore with the Wx interface, run wxstart.exe.

Frequently asked questions

1. When not to use (wx)start.exe?

If you are a developer, and you want to work on OpenKore, then it's highly recommended that you install Perl. OpenKore is able to reload its source code at runtime, so you don't have to restart it every time you made a change. However, this feature is only available if a Perl interpreter is installed. See the Development Introduction Guide for more information about how to install Perl.

2. How do I compile OpenKore?

You don't! In the past, there were two ways to run OpenKore from source:
  1. Install Perl (and necessary third-party Perl modules), and run OpenKore using the Perl interpreter.
  2. Install Perl, and the Perl "compiler" (PerlApp), and compile OpenKore into a .exe file.
Some people choose option 2 because they want to run OpenKore CVS on another computer, and they don't want or can't install Perl. However, both of the above mentioned ways are now obsolete. The only thing you have to do now, is to keep updating the source code. You don't have to compile anything. You don't have to install Perl. You just doubleclick on (wx)start.exe, and that's it.

3. How does this work?

(wx)start.exe is actually a mini-Perl interpreter, which contains all third-party modules that are required to run OpenKore. So this means that:
  • You don't have to recompile (wx)start.exe when the source code changes.
  • You must keep the source code around, otherwise things won't work.
  • Upgrading is simply a matter of upgrading the source code. This saves many hundreds of kilobytes.
