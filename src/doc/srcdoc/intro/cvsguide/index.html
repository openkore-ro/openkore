What is CVS?

The latest public release of OpenKore does not necessarily contain the latest code (the code that developers are working on at this moment). OpenKore has multiple developers, so each developer can't just keep his own code on his own hard drive: he'll get into problems when he wants to merge his changes with other developers' changes.

This is why OpenKore uses CVS (Concurrent Versions System). The latest development source code is stored on the SourceForge CVS server (CVS repository). CVS is a tool that makes it easier for multiple developers to work on one project.

You can compare CVS with an FTP server to which developers upload their changes to. Now you may wonder why we don't just use an FTP server. The reason is, you'll get into trouble when multiple developers work on the same file at the same time. CVS can automatically:

  • Merge your changes with other people's changes.
  • Revert back changes that you've made.
In other words: CVS makes it easier for multiple developers to work on the same project. CVS is widely used in the open source world, and is a very useful tool.

Warning to users

You might be a user who's interested the code in CVS, because it usually has more features. However, the code in CVS might be in development. This means that:
  • The code might be unstable or not well-tested.
  • The code might not work at all.
  • You might need to install additional Perl modules in order to be able to run the code.
In other words: the code in CVS usually also have more bugs. Although we try to keep the code as usable as possible, there's no guarantee that it will work correctly. Please keep this in mind.

When using the code in CVS, you should:

  • Keep a close eye on News.txt.
  • Report any bugs you find, except if News.txt or the forum already says it's a known problem.
