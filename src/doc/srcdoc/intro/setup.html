Setting up the development environment

This section teaches Windows developers how to setup the development environment. If you're a Linux developer you should already have Perl and GCC. All tools listed here are free. You don't necessarily have to use these tools if you prefer other alternative tools, but this guide will only teach you how to setup and use these tools.

ActiveState ActivePerl - the Win32 Perl interpreter

You need a Perl interpreter if you want to work on Kore. The best Perl interpreter for Win32 is probably ActivePerl: http://www.activestate.com/Products/ActivePerl/

Perl is commandline-based. In order to run a Perl script you need to type this in the commandline (assuming ActivePerl is installed to C:\Perl): 

C:\perl\bin\perl YourScript.pl
So, to run openkore.pl, you need to type this: 
cd C:\Whever\Your\OpenKore\Folder\Is
C:\perl\bin\perl openkore.pl

GNU Compiler Collection (GCC) - the C++ compiler

You only need a C++ compiler if you want to work on Tools.cpp and/or Inject.cpp. If you aren't interested in them, you can skip this step.

GCC is a suite which contains a C and C++ compiler. It's originally meant for Unix systems (and is the standard compiler for Linux), but Win32 ports are available. One of the Win32 ports is called MinGW (Minimalist GNU for Win32): http://www.mingw.org/

The OpenKore source code contains the file "Makefile.win32", a makefile for Win32 GCC. You can use that to compile Tools.cpp and Inject.cpp. The file "Makefile" is for Linux/Unix only.

Editors

You can of course edit the source code in any text editor. It is, however, not recommended to use Notepad. Notepad doesn't support CRLF (Unix line endings; at least one of the developers develop OpenKore mainly in Linux) and doesn't support auto-indent.
For Windows, I recommend Crimson Editor, a freeware general-purpose code editor which supports syntax highlighting for many languages (including Perl and C++).
